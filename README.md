## Архитектурные Паттерны
* **Worker Pool (Пул Воркеров):**
    * **`TaskManager`** инициализирует фиксированное число воркеров (**N**) для конкурентной обработки задач.
    * Задания отправляются через буферизованный канал **`TaskQueue`**, что ограничивает одновременную нагрузку.
* **Producer-Consumer:**
    * **HTTP-обработчики** (`/status`, `/report`) выступают в роли **Производителей**, создавая объекты задач и помещая их в **`TaskQueue`**.
    * **Воркеры** (Worker Goroutines) выступают в роли **Потребителей**, извлекая задачи из очереди и выполняя их.
* **Context Cancellation:**
    * Используется **`context.Context`** для установки **таймаутов** на выполнение задач (как индивидуальных, так и общих).
    * Позволяет **отменить** выполнение задачи, если превышено выделенное время, освобождая ресурсы.
* **Graceful Shutdown:**
    * При получении сигналов прерывания (например, `SIGINT` или `SIGTERM`), приложение аккуратно завершает свою работу.
    * Вызываются методы **`tm.Stop()`** (для остановки воркеров) и **`server.Shutdown()`** (для завершения работы HTTP-сервера).
* **Cache (В Памяти):**
    * **`TaskManager`** использует внутренний кэш (**`cacheStatus`**) для хранения последних успешных статусов ссылок.
    * Это позволяет отдавать **быстрые ответы** на повторные запросы, если статус недавно проверялся.
* **Separation of Concerns (Разделение Ответственностей):**
    * Код четко разделен на логические слои: **Контроллеры/Обработчики** (HTTP), **Воркеры** (выполнение задач), **Сервисы** (бизнес-логика, например, генерация PDF).

## API Документация

Сервис предоставляет два основных эндпоинта для работы с задачами: `/status` для мгновенной проверки и `/report` для отчета.

### 1\. Проверка Статуса: `POST /status`

Запускает асинхронную проверку списка ссылок и ожидает результат.

| Параметр | Описание |
| :--- | :--- |
| **Метод** | `POST` |
| **URL** | `/status` |
| **Content-Type** | `application/json` |

#### **Входное Тело (`TaskLinksRequest`)**

```json
{
  "links": [
    "google.com",
    "github.com/...",
    "..."
  ]
}
```

### 2\. Генерация Отчета: `POST /report`

Запускает задачу на проверку ссылок и генерирует на основе результатов PDF-отчет.

| Параметр | Описание |
| :--- | :--- |
| **Метод** | `POST` |
| **URL** | `/report` |
| **Content-Type** | `application/json` |

#### **Входное Тело (`TaskReportRequest`)**

```json
{
  "links_num": [10, 5, 20]
}
```
